# rpc_chanstatsplus

RPC interface for querying stats generated by `chanstats_plus`.

This module exposes methods under the `anope.chanstatsplus.*` prefix via Anope's RPC framework.
It reads from the same SQL table (`<prefix>chanstatsplus`) and does not write stats itself.

Config:

```conf
module {
	name = "rpc_chanstatsplus"
	engine = "mysql/dbstats"
	prefix = "anope_"
	maxlimit = 100
}
```

RPC methods:
- `anope.chanstatsplus.getChannel(channel, [nick], [period], [period_start])`
- `anope.chanstatsplus.getNick(nick, [channel], [period], [period_start])`
- `anope.chanstatsplus.top(channel, [period], [metric], [limit], [period_start])`
- `anope.chanstatsplus.topChannels([period], [metric], [limit], [period_start])`
- `anope.chanstatsplus.topNicksGlobal([period], [metric], [limit], [period_start])`
- `anope.chanstatsplus.listNicksInChannel(channel, [period], [period_start], [limit], [offset])`
- `anope.chanstatsplus.listChannelsForNick(nick, [period], [period_start], [limit], [offset])`

Parameters:
- `period`: `total|monthly|weekly|daily`
- `period_start`: `YYYY-MM-DD` (defaults to the current start of the selected period)
- `metric`: `letters|words|lines|actions|smileys_happy|smileys_sad|smileys_other|kicks|kicked|modes|topics`
- `limit` is clamped to `maxlimit`
